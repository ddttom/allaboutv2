## Usage Notes

### Face Order

The table rows map to cube faces in this order:

1. **Row 1** - Front face (visible at initial -90deg rotation)
2. **Row 2** - Back face
3. **Row 3** - Right face
4. **Row 4** - Left face
5. **Row 5** - Top face
6. **Row 6** - Bottom face

### Image Requirements

- **Format**: JPG, PNG, or WebP
- **Size**: Square images work best (200Ã—200px or larger)
- **Optimization**: Compress images for web performance
- **Aspect Ratio**: 1:1 ratio recommended for best appearance

### Link Guidelines

- Use absolute URLs for external links (https://example.com)
- Use relative URLs for internal navigation (/page/path)
- Ensure all links are valid and functional
- Test navigation on all faces

### Best Practices

1. Use consistent image sizes across all faces
2. Optimize images for web (compress before upload)
3. Provide meaningful alt text for accessibility
4. Test rotation and navigation on all devices
5. Ensure sufficient contrast for hover effects
6. Consider loading state for large images
