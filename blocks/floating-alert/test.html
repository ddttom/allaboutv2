Floating Alert Block - Test Page

About This Test: This page tests the floating-alert block with various configurations. The alert will appear immediately when you load the page. Once dismissed, it won't reappear until you reset the localStorage state using the controls below.

Manual Test Checklist

  • Display Test:
    • Alert appears immediately on page load
    • Modal is centered on screen
    • Overlay covers entire viewport
    • Backdrop blur effect is visible
    • Heading is extracted and styled correctly
    • Horizontal rule separator is visible
    • Content displays below separator
    • Sparkle effects appear periodically
  • Dismissal Methods:
    • Click X button in top-right corner
    • Click outside modal (on overlay)
    • Press Escape key
    • All methods trigger same fade-out animation
    • Modal is completely removed from DOM after dismissal
  • Persistence Test:
    • Dismiss alert
    • Refresh page
    • Alert should NOT reappear
    • Click "Reset & Reload" to see it again
  • Keyboard Navigation:
    • Tab key cycles through focusable elements
    • Shift+Tab cycles backward
    • Focus stays within modal (focus trap)
    • Close button is keyboard accessible
    • Links within content are keyboard accessible
  • Responsive Test:
    • Test on mobile viewport (320px - 480px)
    • Test on tablet viewport (768px - 1024px)
    • Test on desktop viewport (1200px+)
    • Modal remains centered at all sizes
    • Content is readable on small screens
    • Touch targets meet minimum 44x44px
  • Accessibility Test:
    • Use screen reader to verify ARIA attributes
    • Verify role="alert" is announced
    • Check aria-live="polite" behavior
    • Verify close button aria-label
    • Test keyboard-only navigation
    • Check color contrast with tools
  • Animation Test:
    • Fade-in animation is smooth
    • Fade-out animation is smooth
    • Sparkles appear and disappear smoothly
    • No animation jank or stuttering
    • Animations run at 60fps
  • Content Variations:
    • Test with different heading levels (h1-h6)
    • Test without heading
    • Test with one link
    • Test with multiple links
    • Test with long content (scrolling)
    • Test with emojis

Debug Console Commands

Open browser DevTools console and use these commands:

window.floatingAlertDebug.reset() - Clear dismissed state
window.floatingAlertDebug.checkStatus() - Check if dismissed
window.floatingAlertDebug.checkDOM() - Inspect DOM elements
window.floatingAlertDebug.forceShow() - Force display alert

Expected Behavior

  • On Load: Alert appears immediately with fade-in animation
  • Heading Processing: First heading becomes title with separator below
  • Sparkles: Small white sparkles appear randomly every 2 seconds
  • Close Button: X button visible in top-right, accessible by keyboard
  • Overlay Click: Clicking outside modal dismisses it
  • ESC Key: Pressing ESC dismisses modal
  • Fade Out: Dismissal triggers fade-out animation (300ms)
  • Cleanup: Modal removed from DOM, content restored to block
  • localStorage: Dismissed state saved, prevents reappearance
  • Focus Management: Focus moves to modal on display
  • Focus Trap: Tab/Shift+Tab cycles within modal only
