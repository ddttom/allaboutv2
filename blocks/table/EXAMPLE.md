# Table Block - Usage Examples

This document shows how to use the table block in Google Docs to create accessible, responsive tables for data presentation, comparisons, and structured information. Perfect for pricing plans, feature comparisons, specifications, and any tabular data.

## Quick Start

Create a simple table in Google Docs:

| Table |
|-------|
| Feature | Basic | Pro |
| Storage | 5GB | 50GB |
| Users | 1 | 10 |
| Support | Email | Phone |

**Result:** An accessible table with header row (Feature, Basic, Pro) and data rows displaying information in a structured format.

## Basic Usage

### Minimal Example

The simplest table with headers and data:

| Table |
|-------|
| Product | Price |
| Widget A | $10 |
| Widget B | $20 |

**Output:** Two-column table with Product and Price headers.

### Standard Example

Table with multiple columns and rows:

| Table |
|-------|
| Name | Email | Role | Status |
| John Doe | john@example.com | Admin | Active |
| Jane Smith | jane@example.com | Editor | Active |
| Bob Johnson | bob@example.com | Viewer | Inactive |

**Output:** Four-column table with multiple data rows.

## Variations

### Striped Variation

Add alternating row backgrounds for improved readability:

| Table (striped) |
|-----------------|
| Product | Category | Price |
| Laptop | Electronics | $999 |
| Mouse | Accessories | $25 |
| Keyboard | Accessories | $75 |
| Monitor | Electronics | $299 |

**Result:** Odd-numbered rows have a light gray background, making it easier to scan across rows.

### Bordered Variation

Add borders around all cells for clear separation:

| Table (bordered) |
|------------------|
| Country | Capital | Population |
| USA | Washington DC | 331M |
| UK | London | 67M |
| France | Paris | 65M |

**Result:** Full borders around every cell, creating a grid appearance.

### First-line Variation

Highlight the first data row with a pale blue background:

| Table (first-line) |
|--------------------|
| Plan | Price | Features |
| Starter | Free | Basic features only |
| Pro | $29/mo | All features included |
| Enterprise | Custom | Custom solutions |

**Result:** The "Starter" row (first data row) has a pale blue background to draw attention.

Create a table without header row:

| Table (no-header) |
|-------------------|
| Monday | 9:00 AM | Meeting |
| Tuesday | 2:00 PM | Workshop |
| Wednesday | 10:00 AM | Review |

**Result:** All rows treated as data rows, no bold header row.

Mix multiple variations for custom styling:

|---------------------------|

| Apples | 10 | $15 |
| Bananas | 5 | $7 |
| Oranges | 8 | $12 |

**Result:** Table with both alternating row backgrounds AND borders around all cells.

## Content Requirements

### Header Requirements

**Recommended:**
- Clear, descriptive column names
- Keep headers concise (1-3 words)
- Use title case (capitalize first letter)
- Make headers meaningful

**Good headers:**
- Product, Price, Quantity
- Name, Email, Role
- Date, Time, Location

**Avoid:**
- Generic headers (Column 1, Column 2)
- Very long headers (wrap poorly on mobile)
- Special characters (may cause display issues)

### Cell Content Requirements

**Content types supported:**
- Plain text
- Numbers (with or without formatting)
- Currency values ($, €, £)
- Percentages
- Dates and times
- Short phrases

**Content length:**
- Keep cell content concise
- Aim for 1-5 words per cell
- Use abbreviations when appropriate
- Long text may wrap or truncate

**Formatting tips:**
1. Keep numbers aligned (right-aligned looks best)
2. Use consistent date formats
3. Include units where applicable (GB, lbs, inches)
4. Use abbreviations for long text

### Row Requirements

**Minimum requirements:**
- At least 1 header row + 1 data row
- All rows should have same number of columns
- Empty cells are okay (just leave blank)

**Recommended:**
- 2-15 data rows (optimal readability)
- More than 20 rows may need pagination
- Keep column count to 3-8 for mobile

## Styling Guidelines

### Visual Consistency

**Default table appearance:**
- Header row: Bold text, top and bottom borders
- Data rows: Regular text, bottom borders only
- Cell padding: 8px 16px
- Font size: Responsive (smaller on mobile)

**Variations enhance appearance:**
- **Striped**: Adds alternating row backgrounds
- **Bordered**: Adds borders around all cells
- **First-line**: Highlights first data row
- **No-header**: Removes header row styling

### Responsive Behavior

**Mobile (< 600px):**
- Tables scroll horizontally if too wide
- Font size reduced for readability
- Touch-friendly scrolling
- Cell padding maintained

**Tablet (600px - 899px):**
- Increased font size
- Most tables fit without scrolling
- Comfortable reading experience

**Desktop (≥ 900px):**
- Largest font size
- Tables typically fit full width
- Optimal readability

**How it works:**
- Wide tables scroll horizontally (don't wrap)
- Table structure maintained at all sizes
- No column hiding or stacking (preserves data)

### Color and Theming

**Default colors:**
- Striped rows: Light gray (rgba(0, 0, 0, 0.05))
- First-line: Pale blue (#e6f2ff)
- Borders: Inherit from site theme

**Customization:**
- Colors controlled via CSS variables
- Can be customized per site theme
- Supports dark mode with custom CSS

## Best Practices

### Content Strategy

**1. Organize logically**
- Put most important data in first columns
- Order rows by importance or logical flow
- Group related information together
- Consider sort order (alphabetical, numerical, chronological)

**2. Keep headers concise**
- 1-3 words per header
- Clear and descriptive
- Avoid abbreviations unless widely known
- Use consistent naming conventions

**3. Balance data density**
- Not too sparse (wastes space)
- Not too dense (hard to read)
- Aim for scannable content
- Use white space effectively

**4. Consider mobile users**
- Limit columns (3-5 ideal for mobile)
- Keep cell content short
- Test horizontal scrolling
- Prioritize important columns first

### Accessibility First

**1. Always include headers**
- Use default table (not no-header) when possible
- Headers provide context for screen readers
- Column headers help users understand data

**2. Write clear headers**
- Descriptive, not generic
- "Product Name" not "Name"
- "Monthly Price" not "Price"
- Clear context for all users

**3. Keep content readable**
- Plain language in cells
- Avoid jargon or abbreviations
- Spell out units (GB not G)
- Use consistent formats

**4. Test with screen readers**
- Headers announced before cell content
- Row and column navigation works
- Cell relationships clear

### Performance Optimization

**Keep tables manageable:**
- Under 100 rows (no performance issues)
- Under 10 columns (mobile friendly)
- Avoid huge tables (use pagination)
- Split large datasets into multiple tables

**Content loading:**
- All table data loaded with page
- No lazy loading needed for small tables
- Fast rendering (minimal JavaScript)

### When NOT to Use Tables

**Avoid tables for:**
- Single-row data (use list or paragraph)
- Non-tabular content (use appropriate block)
- Complex hierarchies (use nested lists)
- Visual layout only (use CSS grid/flexbox)

**Better alternatives:**
- **Cards** - For visual content browsing
- **List** - For simple sequential data
- **Accordion** - For expandable content sections
- **Columns** - For side-by-side layout

## Troubleshooting

### Issue: Table too wide on mobile

**Problem:** Table extends beyond screen on mobile devices.

**Solution:**
- This is expected behavior for wide tables
- Users can scroll horizontally to view all columns
- Consider reducing columns for mobile version
- Prioritize important columns in leftmost positions

### Issue: All rows look the same (striped not working)

**Problem:** Striped variation not showing alternating backgrounds.

**Cause:** Variation not specified in markdown.

**Solution:**
- Ensure first cell says `Table (striped)`
- Check spacing: `Table (striped)` not `Table(striped)`
- Verify no typos in variation name

### Issue: No bold headers

**Problem:** Header row looks like regular data row.

**Cause:** Using no-header variation or missing header row.

**Solutions:**
1. Remove `(no-header)` if used by mistake
2. Ensure first row is intended as headers
3. Check if first row was accidentally marked as data

### Issue: Borders missing (bordered variation)

**Problem:** Bordered variation not showing cell borders.

**Cause:** Variation not specified correctly.

**Solution:**
- Use `Table (bordered)` in first cell
- Check spelling and spacing
- Verify CSS is loaded

### Issue: Headers wrapping on mobile

**Problem:** Long header text wraps to multiple lines on mobile.

**Solutions:**
1. **Shorten headers** (best solution):
   - "Product Name" → "Product"
   - "Monthly Price" → "Price"
   - "Description" → "Details"

2. **Accept wrapping** (acceptable):
   - Headers wrap to maintain readability
   - This is normal behavior

3. **Use abbreviations**:
   - "Quantity" → "Qty"
   - "Description" → "Desc"
   - "Reference" → "Ref"

### Issue: Empty cells showing oddly

**Problem:** Empty cells look different from filled cells.

**Solution:**
- This is normal behavior
- Empty cells are valid
- If needed, use dash (-) or "N/A" as placeholder

## Testing Your Tables

### Visual Testing Checklist

After creating table in Google Docs:

1. **Preview on staging site:**
   - Table displays correctly ✓
   - Headers are bold ✓
   - Data rows formatted properly ✓
   - Borders show correctly ✓

2. **Check variations:**
   - Striped rows alternate ✓
   - Bordered cells have borders ✓
   - First-line highlighted (if used) ✓
   - No-header works (if used) ✓

3. **Mobile testing (< 600px):**
   - Table scrolls horizontally ✓
   - Text remains readable ✓
   - Cell content not cut off ✓
   - Touch scrolling works ✓

4. **Accessibility testing:**
   - Screen reader announces headers ✓
   - Cell content associated with headers ✓
   - Keyboard navigation works ✓
   - Table structure clear ✓

### Browser Testing

Test in multiple browsers:
- Chrome/Edge (last 2 versions)
- Firefox (last 2 versions)
- Safari (last 2 versions)
- iOS Safari (mobile)
- Android Chrome (mobile)

**Not supported:** Internet Explorer 11
