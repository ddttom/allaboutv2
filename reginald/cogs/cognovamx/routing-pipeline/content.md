# Routing Pipeline

Preprocess user prompts with spell correction, route matching, and context injection. Cut inference cost by telling Claude where to look before it starts searching.

## The Problem

Every prompt costs inference. Complex prompts waste compute because Claude searches for context that already exists in structured ROUTING.cog.md files. Tom types fast with typos, which costs additional inference to interpret.

## The Solution

A preprocessing pipeline that runs before Claude sees the prompt:

1. **aspell** fixes obvious typos locally (zero inference cost)
2. **Smart trigger** skips simple prompts (no latency penalty)
3. **Route matching** finds relevant folders from the routing registry
4. **Context injection** tells Claude exactly where to look

## Components

| Component | Path | Purpose |
|---|---|---|
| This SOP | `scripts/cogs/routing-pipeline.cog.md` | The instruction |
| Hook | `.claude/hooks/route-decorator.sh` | The runtime |
| Sync script | `scripts/route-sync.js` | Registry generator |
| Registry | `mx-canon/mx-maxine-lives/routing-registry.json` | Auto-generated index |

## How It Works

The hook fires on every `UserPromptSubmit` event. It reads the prompt, runs the pipeline, and outputs routing context as `additionalContext` in the hook response JSON. Claude receives both the original prompt and the routing context.

The routing registry is auto-generated by `npm run route:sync`. It scans all ROUTING.cog.md files in mx-canon, extracts intents and keywords, and writes a JSON index. Every initiative with a ROUTING.cog.md is automatically included.

## Dependencies

- **aspell** — `brew install aspell` (optional, degrades gracefully)
- **jq** — `brew install jq` (required for JSON parsing in hook)
- **js-yaml** — npm devDependency (required for sync script)
- **Node.js** — Required for sync script
