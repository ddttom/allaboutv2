About This Book

Modern websites work beautifully for humans but fail completely for AI agents. This book examines how interfaces optimized for human visual browsing break for automated visitors — and shows that fixing this benefits everyone.

The patterns that break AI agents also break humans. Toast notifications that vanish, pagination hiding content, single-page applications with invisible state changes — these have been creating accessibility problems for years. Now AI agents are struggling with the same patterns, and there's commercial pressure to fix them.

January 2026 changed everything. In seven days, Amazon, Microsoft, and Google launched AI agent commerce systems. The shift from future concern to present reality happened faster than anyone expected. This book examines the collision between agent capabilities and web design conventions — and provides practical solutions for adapting to the platform race.

Key Themes

Agent Diversity and Universal Patterns

The book addresses a diverse ecosystem of AI agents with varying capabilities:

  • CLI agents: Command-line tools running locally (Claude Code, Cline)
  • Local (SMOL) agents: Lightweight agents on user devices
  • Server-based agents: Cloud-hosted agents accessing remotely
  • Browser agents: Full browser automation (Playwright, Selenium)
  • Browser extension assistants: In-browser AI tools
  • IDE-integrated browser controls: Development environments with browser integration

Universal Compatibility Patterns

The patterns aim for universal compatibility:

  • Semantic HTML: Works for all agents, regardless of JavaScript execution
  • Explicit state attributes: Visible in DOM for any parser
  • Structured data: Machine-readable regardless of agent architecture
  • Clear feedback: Persistent and unambiguous for all interaction models

Session Inheritance Problem

Key security insight from Chapter 6: In-browser agents inherit authenticated sessions rather than failing to authenticate. Banks cannot detect AI involvement because agents inherit proof-of-humanity tokens from the authenticated user's session.

The Platform Race

January 2026 brought unprecedented urgency to AI agent compatibility:

  • Seven-day acceleration: Amazon (Alexa+), Microsoft (Copilot Checkout), and Google (UCP/Business Agent) launched agent commerce systems within one week
  • Open vs closed protocols: Google's Universal Commerce Protocol (UCP) and OpenAI's Agentic Commerce Protocol (ACP) compete against Microsoft's proprietary system
  • Commercial urgency: Timeline compressed from "12-18 months" to "happening now"
  • Competitive landscape: Platforms betting billions that AI agents will mediate how humans shop online

Standards Classification

All technical patterns are clearly labeled by maturity status:

  • Established Standards: Schema.org, semantic HTML, ARIA - use with confidence
  • Emerging Conventions: llms.txt - early adoption phase
  • Proposed Patterns: ai-* meta tags, data-agent-visible - not yet standardized
  • Speculative: Future possibilities, clearly marked as such

All proposed patterns are forward-compatible (won't break if agents don't recognize them).
